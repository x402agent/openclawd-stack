## Highlights

### Token Program Auto-Detection

`OnlinePumpSdk.fetchBuyState()` and `fetchSellState()` now **automatically detect** whether a mint uses SPL Token or Token-2022 by inspecting the mint account owner on-chain. The `tokenProgram` parameter is now optional â€” existing callers are unaffected.

Both methods now return `tokenProgram` in their result object, making it easy to pass through to instruction builders.

### Convenience Buy/Sell Wrappers

New `buyInstructions()` and `sellInstructions()` methods on `OnlinePumpSdk` combine state fetching (global config) with instruction building in a single call, reducing boilerplate for common trading flows.

### PumpKit Framework Expansion

The `pumpkit/` monorepo received a major expansion:

- **@pumpkit/web** â€” Full React dashboard with Dashboard, CreateCoin, Docs, Home, Profile, and Token pages
- **useEventStream hook** â€” SSE connection management with status tracking
- **26 new tutorials** (tutorials 20â€“45) covering MCP server, WebSocket feeds, channel bot, mayhem mode, DeFi agents, analytics, and more
- **CI workflow** with automated testing
- **Fee Distribution Monitor** for WebSocket-based distribution event detection
- **Components:** WatchForm, WatchList, StatsBar, EventCard, CopyButton, charts with cleanup/resize handling

### Lair-TG â€” Unified Telegram Bot Platform

New `lair-tg/` directory scaffolds a unified Telegram bot platform for DeFi intelligence, with config, data sources, formatters, health checks, logging, and type definitions. Includes Dockerfile and Railway deployment config.

### Channel-Bot: Graduation Events

The channel bot now supports **graduation events** with enriched token data including holders, recent volume, replies, and KotH timestamps. Also added token disambiguation by market cap and same-name token detection.

### Claim-Bot: Social Fee Claims

Major claim monitoring overhaul:
- **ClaimMonitor** and **RpcClaimMonitor** for tracking fee claims via WebSocket and direct RPC
- GitHub social fee claims with first-claim badges, lifetime tracking, and fake claim detection
- Rich claim cards with token images, creator profiles, and developer enrichment
- Social-fee-index for contract address resolution

### X/Twitter Integration Overhaul

Migrated from the deprecated Twitter API v2 to Twitter's internal **GraphQL API** with cookie-based authentication via `xactions`. Added `createdAt`/`tweetCount` fields and influencer tier logic.

### PumpOS Website

New `website/` directory with a full HTML/CSS/JS PumpOS web desktop, deployed via Vercel.

## Upgrade Guide

```bash
npm install @nirholas/pump-sdk@1.31.0
```

No code changes required. If you were explicitly passing `TOKEN_PROGRAM_ID` to `fetchBuyState` / `fetchSellState`, you can now omit it for automatic detection.

**New convenience pattern:**
```typescript
const online = new OnlinePumpSdk(connection);
const buyState = await online.fetchBuyState(mint, user);
// buyState.tokenProgram is auto-detected
const ixs = await online.buyInstructions({
  ...buyState, mint, user, amount, solAmount, slippage: 1,
});
```
